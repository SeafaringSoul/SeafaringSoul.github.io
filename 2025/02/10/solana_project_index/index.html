账户

参考文档：https://solanacookbook.com/zh/core-concepts/accounts.html

在Solana中有三类账户：

  • 数据账户，用来存储数据

  • 程序账户，用来存储可执行程序（程序编译后会自动生成）

  • 原生账户，指Solana上的原生程序，例如”System”，”Stake”，以及”Vote”。

  • 平时我们接触的比较多的是前两种，其中数据账户又分为两类：

  • 系统所有账户（也就是我们自己用的账户）

  • 程序派生账户（PDA）（基于自己账户派生出的新账户）

每个账户都有一个地址（通常是一个公钥）以及一个所有者（程序账户的地址）。下面是账户存储的完整字段列表：

字段 描述
lamports 账户拥有的 lamport（Solana 最小单位）数量
owner 账户的所有者程序
executable 账户是否可以处理指令
data 账户存储的数据字节码
rent_epoch 下一个需要付租金的 epoch（代）

Solana 程序

任何开发者都可以在 Solana 链上编写并部署程序。Solana 程序（在其他链上称为智能合约）是所有链上活动的基础。
📖 参考文档：Solana Cookbook

Solana 程序架构

Solana 程序通常使用 Rust 编写，常见的架构如下：

文件 描述
lib.rs 注册模块
entrypoint.rs 程序的入口点
instruction.rs 程序 API，处理指令数据的序列化与反序列化
processor.rs 业务逻辑
state.rs 程序对象，处理状态的反序列化
error.rs 程序自定义的错误

Solana 支持的环境及 RPC 连接

集群环境 RPC 连接 URL
Mainnet-beta https://api.mainnet-beta.solana.com
Testnet https://api.testnet.solana.com
Devnet https://api.devnet.solana.com
Localhost 默认端口：8899（例如 http://localhost:8899）
